# Plan Sync Skill

統一管理 `plan.md` 的 Skill，讓 Antigravity 和 Claude Code 協作時任務狀態保持同步。

**根據使用者說的關鍵字判斷模式：**

| 使用者說的 | 觸發模式 |
|-----------|---------|
| 計畫、規劃、拆解、新增任務、設計 | 規劃模式 → 寫入 plan.md |
| 執行、實作、開始做、繼續、寫程式 | 實作模式 → 讀取並執行 |

不管是 Antigravity 還是 Claude Code，任何工具都照這個規則走。

---

## plan.md 標準格式

```markdown
# 專案任務計畫

> 規則：
> - 規劃角色負責新增任務（- [ ]）
> - 實作角色負責勾選完成（- [x]）並移至已完成區塊
> - 進行中超過 10 項時請封存

## 進行中
- [ ] 任務描述（具體、可執行）

## 已完成
- [x] 已完成的任務
```

### 狀態標記
| 標記 | 意思 | 誰來改 |
|------|------|--------|
| `- [ ]` | 待實作 | 規劃角色新增 |
| `- [x]` | 已完成 | 實作角色更新 |

---

## 規劃模式（Antigravity）

收到規劃需求時執行以下步驟：

**1. 讀取或建立 plan.md**

若不存在，建立標準格式的空白 plan.md。

**2. 拆解任務**

將功能拆解為具體子任務，每個任務要：
- 單一職責（一個任務只做一件事）
- 具體明確（避免「處理 API」，改用「串接 /api/login 並處理 401 錯誤」）
- 可在單次實作 session 內完成
- 有相依關係時在描述中標註（例如：「[需先完成登入] 實作個人頁面」）

**3. 新增到「進行中」區塊末尾**

**4. 絕對不可以：**
- 修改或刪除任何 `- [x]` 項目
- 動到「已完成」區塊

**5. 封存提醒**

當「進行中」超過 10 項時主動提醒：
```
⚠️ 進行中任務已超過 10 項，建議封存已完成的任務。要我執行封存嗎？
```
若使用者同意，將所有 `[x]` 從「進行中」移到「已完成」。

---

## 實作模式（Claude Code）

被要求執行任務時：

**1. 讀取 plan.md**

只看「進行中」區塊，忽略「已完成」區塊。

**2. 確認任務**

```
我將開始實作：「[任務名稱]」
預計修改的檔案：[列出相關檔案]
開始嗎？
```

**3. 實作**

- 專注在當前單一任務
- 不要順便做其他任務
- 遇到問題先暫停回報

**4. 完成後更新 plan.md**

- 把完成的 `[ ]` 改成 `[x]`
- 將該項目移到「已完成」區塊

**5. 回報結果**

```
✅ 完成：「[任務名稱]」
修改了：[列出檔案]

進行中還有 [N] 個任務，要繼續下一個嗎？
```

**絕對不可以：**
- 新增任何新任務
- 修改任務描述
- 刪除任何任務

---

## 遇到問題時

**任務無法完成：**
```
⚠️ 無法完成「[任務名稱]」
原因：[說明問題]
需要：[說明需要什麼]
要跳過還是提供更多資訊？
```

**任務有相依問題：**
```
⚠️ 「[任務名稱]」依賴「[另一個任務]」尚未完成
建議先實作「[另一個任務]」，要調整順序嗎？
```
